Homework 5 - Matrices, Filters, and Guidance

Due on 2020-04-03 at class time

Purpose

The purpose of this assignment is to apply an inheritance model to the problem of filtering and state estimation, and to investigate the numerical constraints of different filtering methods. Specifically, you will use C++ to explore the following areas of interest:

  • Matrix operations, e.g. inversion and multiplication
  • Abstract classes/interfaces
  • Inheritance and polymorphism
  • Matrix representations of dynamic systems
  • Templated functions and objects
  • Interactions between objects of differing classes

Background

In the previous assignment you implemented a closed-loop control system, in which the control signal was dependent on the feedback from the model (the blood glucose concentration). It was therefore assumed that the input to the controller exactly matched the state of the model. Real-life implementations of control systems cannot safely make this assumption - they rely on sensors whose measurements are often imprecise. In this assignment you will be responsible for implementing part of the control system used by the Apollo program to safely land astronauts on the moon.

For purposes of simplicity, we will be focusing only on attitude control in two dimensions, i.e., the angle of the lunar lander with respect to the ground normal (the y-axis in a conventional coordinate system).

Consider the following diagram:

Note the application of a force FF (from a rocket thruster) perpendicular to the axis of rotation at a distance RR from the lander’s center of gravity.
We can then use equations from your Physics I course to solve for torque:

τ=R×Fτ=R×F
RR and FF are perpendicular so their cross product is simply their product:
τ=RFτ=RF
We can then solve for angular acceleration, αα:
τ=Iατ=Iα
Rearranging terms:
α=τIα=τI
We can then substitute our value of ττ:
α=RFIα=RFI

The state of the system is represented by αα and its single and double integrals with respect to time, ωω and θθ, which correspond to angular velocity and angle, respectively.

As with the previous assignment, you will use a PID controller - this time to control the thruster force FF to reach a desired value of θθ.

But a PID controller alone is not sufficient in this case - the sensors used to measure state, as with any real-world sensors, are prone to noise. Note the noise observed by the gyroscopes on the actual Apollo 8 mission:

To effectively control a system with noisy measurements, the noise must be filtered out. You will use inheritance and polymorphism to implement an interface base class and subsequently several derived classes, that represent the types of filters discussed in class, namely:

  • a dummy filter (which is just a pass-through)
  • the single-pole low-pass filter (exponential moving average)
  • the Kalman filter

You will also implement a combination filter that allows for encapsulating and chaining multiple filters. This filter should accept an arbitrary number of filters (using a data structure of your choice) and should filter a noisy input by sequentially applying these filters. For an example, see the driver instructions.

Additional Background Information

The Kalman filter’s first application was to remove noise from Apollo Lunar Module sensor data and thus to improve the accuracy of the guidance logic. You will apply your filters to a simplified version of this problem. You can read more about the Kalman filter’s introduction in this NASA paper, or read through the source code (written in AGC assembly), but I don’t think you’ll find it particularly helpful for this assignment.

Program Specification

It is recommended that you start by implementing your matrix class, then your model class (which should be similar to previous assignments), and finally your “real” filter classes.

In addition to the three filters described above, you will implement a dummy filter that returns the noisy measurements without changing them so that you can test your model, and later to evaluate the performance of your real filters.

Your lunar lander class must implement the following:

  • Appropriate constructors/destructors/copy functionality
  • Function evaluation operator () that “steps” the model, calculating a new state given the thruster force input
  • A means of writing the current state of the model to a std::ostream (stream operator), using the format Alpha: <alpha>, Omega: <omega>, Theta: <theta>
  • A means of interacting with a PID controller to send the current state and receive the thruster force
  • A means of using any filter derived from your filter interface base to filter the “measured” state prior to sending it to the controller

Since there are no real sensors involved with this model, you should also add Gaussian noise to the calculated state before sending it to the filter. You should use the utilities available in the C++ standard library to add noise in the range between zero and ±25±25 percent relative to each element of your raw (not-noisy) state.

You will also use matrices for your filter implementations, in particular the Kalman filter. Your matrix class, which should be templated to allow for matrices of any appropriate type, should include the following:

  • A constructor accepting a two-dimensional std::initializer_list
  • Appropriate constructors/destructors/copy functionality
  • Copy constructor/copy assignment operations
  • Element access with the [] operator (optional)
  • Element access with the () operator (required), such that matrix(i,j) can be used to access the element in row i and column j
  • A means of determining the number of rows and columns with rows() and cols() functions, respectively
  • Matrix addition, matrix subtraction, matrix multiplication, scalar multiplication, and matrix-by-vector multiplication
  • Unary minus
  • Inversion via the Newton-Schulz method discussed in class with an invert function
  • Stream operators for input/output of matrix data - as with your vector, you should be able to read in an arbitrarily-sized matrix from an istream, in which rows are delimited by newlines and columns are delimited by spaces, and output should be in the same format (the final row should not have a trailing newline)

As with your vector class, all matrix arithmetic-related operations should return a new matrix. Your matrix class should also not use std::vector or std::valarray.

You will also use inheritance to model the relationships between different types of filters, all of which do the same thing (filter a noisy signal), but have very different implementations. The core requirements for your filters are as follows:

  • overload of function evaluation operator () to return a filtered signal given a noisy signal (system state), and any other parameters you deem necessary As with previous assignments, your filters must be templated on the type used to represent the state that is being estimated. You should assume that the state can always be represented by an instance of your vector class, but that the thing stored inside the vector may change. That is, if your filter is templated on T, it means that your state can be represented by a yourVectorClass<T>.

Your filters should be configurable. That is, hardcode as little as possible. For maximum utility your filters should be adaptable to problems other than the one presented in this assignment. For example, the state transition matrices within your Kalman filter should not be hardcoded.

Some hints with regard to your Kalman filter implementation:

  • Recall that the measurements from the system (e.g. the current angle of the lunar lander) are also the state of the system
  • You may assume that process error is zero, because the equations used to model state transitions perfectly characterize the system
  • You should experiment with different values of measurement error, but start off with the identity matrix multiplied by a scalar value in the range (0.1,0.5)(0.1,0.5)

The internal implementation of the filters will differ significantly, but you should make an effort to make as much of the external interface purely virtual as possible, i.e. part of your filter abstract class.

You should also pay attention to the performance of your filters. Some of the mathematical operations required are computationally expensive, and as a result you should ensure that these operations are optimized where possible and not performed unnecessarily.

As always, you are expected to follow the assignment guidelines.
